# governance/

Project governance, active epics, architecture documentation, and ticket tracking.

## Key documents

| File | Purpose |
|---|---|
| `TICKETS_MASTER.md` | Single source of truth for all active tickets across the project |
| `AEGIS_CONTEXT.md` | Architecture context map for agents and contributors |
| `AEGIS_MASTER_CODEX.md` | Universal rules for all AI agents |
| `EPIC_*.md` | Epic-level planning and design documents (e.g., `EPIC_53_STABILIZATION.md`) |
| `ADR-*.md` | Architecture Decision Records (e.g., `ADR-CAA-015.md`) |
| `IDEA_*.md` | Idea drafts and concepts (e.g., `IDEA_AEGIS_CONNECT.md`) |

## Tickets/

Individual ticket files. One file per ticket. Format: `<ID>.md`

Active Ticket namespace:
- `CORE-*` — unified ticket identifier across all components (kernel, shell, app, installer, devops, docs, etc.)
- `EPIC_*` — epic-level planning documents
