# governance/

Project governance, architecture documentation, and ticket tracking.

## Key documents

| File | Purpose |
|---|---|
| `TICKETS_MASTER.md` | Single source of truth for all tickets across the project |
| `AEGIS_CONTEXT.md` | Architecture context map for agents and contributors |
| `AEGIS_MASTER_CODEX.md` | Universal rules for all AI agents |
| `AUDIT_REPORT.md` | Security audit findings and resolutions |
| `TEAM_WORKFLOW.md` | How the hybrid human-AI team works |

## Tickets/

Individual ticket files. One file per ticket. Format: `<ID>.md`

Ticket IDs by area:
- `ANK-*` — kernel work
- `SH-*` — shell/UI work
- `APP-*` — mobile app work
- `INST-*` — installer/devops work
- `GOV-*` — governance/documentation work
- `EPIC_*` — epic-level planning documents
